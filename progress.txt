CURRENT STATUS: PHASE 18 — MAJOR ARCHITECTURAL OVERHAUL COMPLETE
================================================================================

Complete pipeline redesign. Old 3-model council replaced with a
Consultant → Reviewer → Chairman → Step-by-step generation architecture.

Pipeline flow (new):
  Input (text + images)
  -> Consultant (Gemini) — chat loop, bay-by-bay analysis, two modes
  -> User confirms with "yes/confirm"
  -> Council Reviewer (GPT) — checks summary against image for missed details
     -> If issues: back to Consultant → user resolves → confirm again
     -> If approved: Chairman generates prompts
  -> Chairman (Gemini) — generates 3 clean image prompts
  -> Floor Plan generated → user approves (or requests modification)
  -> Front Elevation generated (floor plan as reference) → user approves
  -> 3D Realistic Render generated (both as references) → user approves
  -> Gallery
  Modifications: return to Consultant conversation → re-review → regenerate step

================================================================================
WHAT EACH SYSTEM DOES (Current State)
================================================================================

1. OpenRouter Client (app/services/openrouter.py):
   Async HTTP client using httpx. Supports text chat completions,
   vision (image+text), multi-image vision, and image generation.
   Has retry logic with exponential backoff for rate limits (429)
   and server errors (5xx). Includes JSON extraction from responses
   (handles raw JSON, markdown code blocks, embedded JSON).
   run_async() bridges async code into Streamlit's sync context.

2. Design Specification Document — DSD (app/models/dsd.py):
   Pydantic model that captures all design details: type, name,
   description, dimensions, style, materials, colors, structural
   elements, spatial layout, context, views to generate, and
   change history. Supports versioning via create_new_version().
   Has to_prompt_description() for feeding into generation prompts.
   Has get_applicable_views() to determine which views make sense
   for a given design type (e.g., furniture skips rear elevation).

3. Project Model (app/models/project.py):
   Tracks project lifecycle from DRAFT through COUNCIL_REVIEW,
   AWAITING_CONFIRMATION, GENERATING, QUALITY_REVIEW, COMPLETE,
   to REFINING. Stores input data, generated images with metadata,
   and DSD version history.

4. Council System (app/agents/council.py + chairman.py):
   Full 3-round deliberation:
   - Round 1: Each member independently analyzes input (parallel async)
   - Round 2: Each member reviews others' interpretations (parallel)
   - Round 3: Each member produces final converged interpretation (parallel)
   - Consensus check (basic: 2/3 valid responses = consensus)
   - Chairman synthesizes into a DSD
   Supports text-only, image-only, and mixed (text+image) input.
   Progress callback for UI updates.

5. Council State (app/models/council_state.py):
   Tracks all deliberation rounds, member responses (text + structured
   JSON), consensus status, and chairman decisions. Full audit trail.

6. Prompt Templates:
   - Council: System prompt, Round 1 (text/image/mixed variants),
     Round 2 (cross-review), Round 3 (convergence), Chairman synthesis
   - Generation: Per-view prompts (floor plan, 3 elevations, 2 3D perspectives)
   - Review: Quality scoring (5 criteria), change verification
   - Refinement: Change classification, DSD update, targeted regen

7. Services:
   - ImageService: Load/encode/resize images for API, decode/save
     generated images, create thumbnails
   - ProjectStore: JSON file-based persistence for projects and DSDs.
     Versioned DSD storage (dsd_current.json + dsd_v{n}.json).
     Project listing, deletion, image management.

8. Streamlit UI (app/ui/main.py):
   Multi-page app with sidebar navigation:
   - New Project: Upload sketch or type description (3 tabs)
   - Council View: Shows deliberation progress and details
   - DSD Review: Formatted display of the agreed specification
   - Gallery: Image grid, download, ZIP, per-view regen
   - Quality Review: Scores, feedback, approval, re-review
   - Refinement: Change request (placeholder, Phase 6)
   - Projects: List all projects with open/delete

NEW CONSULTANT (app/agents/consultant.py + app/prompts/consultant_prompts.py)
-------
Two modes, auto-detected by the model from context:

Mode 1 — Image Analysis:
  - Splits design into numbered bays (left to right), walls for multi-wall layouts
  - States only what it is CONFIDENT about; asks about the rest
  - Gives 2-3 examples when asking about visual/decorative details
  - Checks: top trim/cornice, side columns thickness, grooves, door types, hardware,
    shelf positions, bay alignment, base treatment, layout
  - Asks explicit layout question (which bays are on same wall, facing each other, etc.)
  - Loops until user confirms

Mode 2 — Participant Design:
  - Acts as a design partner making specific suggestions
  - Checks practicality, proportions, usability
  - Adds design insights when relevant
  - Loops until user confirms

Output is always JSON: {"response": "...", "status": "chat"|"confirmed", "final_summary": "..."}
When status == "confirmed", final_summary is the complete structured design description.

NEW COUNCIL REVIEWER (app/agents/council.py + app/prompts/council_prompts.py)
-------
Single GPT model that reviews the consultant's Final Design Summary against
the original image(s). Checks specifically for:
  - Top trim/cornice type
  - Side column details and grooves
  - All bay door/opening types
  - Hardware type and placement
  - Shelf positions inside open sections
  - Bay alignment (height differences)
  - Base treatment
  - Any detail visible in image but missing from summary
  - Layout clarity

Returns: {"approved": bool, "issues": ["one-liner per issue"]}
If issues: Consultant reformulates them as natural questions to the client,
  then the loop continues. If approved: Chairman generates prompts.

NEW CHAIRMAN (app/agents/chairman.py)
-------
Receives full conversation. Generates 3 clean, direct image prompts:
1. Floor plan: top-down 2D, B&W, lines only, dimensions only if given
2. Front elevation: left to right bay-by-bay, B&W, 2D
3. Realistic render: photorealistic 3D, materials, natural lighting

Prompts are printed to terminal for developer review before generation.

NEW GENERATOR (app/agents/generator.py)
-------
New method generate_step(prompt, view_type, project_id, reference_images).
No DSD dependency. Step-by-step:
  - floor_plan: no references
  - front_elevation: floor plan as reference
  - realistic_render: floor plan + front elevation as references
Reference images are read back from disk after generation and passed to next step.
openrouter.generate_image updated to support multiple reference_images.
